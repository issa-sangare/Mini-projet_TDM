Les différentes techniques de codage de signal ont leurs propres avantages et inconvénients en termes de synchronisation, de sensibilité au bruit et de bande passante. Voici une comparaison générale :

### Meilleurs codes pour la synchronisation :
- **Manchester différentiel (Manchester Differential)** : Il est souvent utilisé pour sa capacité intrinsèque de synchronisation, car il garantit une transition de signal à chaque bit, facilitant ainsi la synchronisation du récepteur.
- **Miller** : Il permet également une bonne synchronisation car il maintient un niveau de signal constant pendant la transmission de données, ce qui peut être plus facile à synchroniser.

### Codes moyens pour la synchronisation :
- **NRZ (Non-Return-to-Zero)** : La synchronisation peut être plus complexe car il n'y a pas de transition de signal à chaque bit, ce qui nécessite des techniques de synchronisation supplémentaires.
- **AMI (Alternate Mark Inversion)** : Bien qu'il puisse être utilisé avec des techniques de synchronisation, il peut nécessiter des mécanismes supplémentaires pour la synchronisation précise des données.

### Pires codes pour la synchronisation :
- **NRZI (Non-Return-to-Zero Inverted)** : Comme il dépend des transitions de signal pour représenter les bits, la synchronisation peut être plus difficile, en particulier en présence de bruit ou de distorsion du signal.
- **CMI (Coded Mark Inversion)** : Il peut être plus difficile à synchroniser en raison de la nature de son codage, nécessitant souvent des techniques de synchronisation avancées.

### Sensibilité aux bruits :
- Les codes qui maintiennent une transition de signal à chaque bit (comme Manchester) ont tendance à être moins sensibles aux bruits, car les transitions de signal permettent au récepteur de récupérer plus facilement le signal même en présence de bruit.
- Les codes qui ne maintiennent pas de transition de signal à chaque bit (comme NRZ) peuvent être plus sensibles aux bruits, car il peut être plus difficile pour le récepteur de distinguer les bits en présence de bruit.

### Bande passante :
- Les codes qui nécessitent plus de transitions de signal par bit (comme Manchester) peuvent avoir une bande passante plus large en raison de la fréquence plus élevée des transitions.
- Les codes qui maintiennent un niveau de signal constant pendant la transmission de données (comme Miller) peuvent avoir une bande passante plus faible car ils nécessitent moins de transitions de signal.

Il est important de noter que le choix d'un code dépend souvent des exigences spécifiques de la communication, y compris la bande passante disponible, la robustesse contre les bruits et les contraintes de synchronisation.

- **Utilisation** : NRZ est l'un des types de codage de signal les plus simples où un niveau de tension constant représente un bit "1" et un niveau de tension nul représente un bit "0". Il est largement utilisé dans les communications numériques terrestres et spatiales.

### NRZI (Non-Return-to-Zero Inverted) :
- **Utilisation** : NRZI est similaire à NRZ, mais au lieu de représenter les bits directement par le niveau de tension, il utilise les transitions de signal pour indiquer les bits. Il est souvent utilisé dans les interfaces série telles que RS-232.

### Manchester :
- **Utilisation** : Manchester est utilisé pour garantir une transition de signal à chaque bit, ce qui facilite la synchronisation. Il est couramment utilisé dans les communications Ethernet, ainsi que dans d'autres protocoles de communication série.

### Manchester différentiel :
- **Utilisation** : Comme Manchester, il garantit une transition de signal à chaque bit, mais utilise également des transitions supplémentaires pour représenter les bits. Il est souvent utilisé dans les réseaux locaux (LAN) pour sa robustesse contre les interférences.

### AMI (Alternate Mark Inversion) :
- **Utilisation** : AMI est utilisé pour minimiser la composante DC du signal tout en fournissant un moyen de synchronisation des bits. Il est utilisé dans diverses applications, y compris les communications numériques terrestres et les liaisons T1/E1.

### MLT3 (Multi-Level Transition 3) :
- **Utilisation** : MLT3 utilise plusieurs niveaux de tension pour représenter les bits, ce qui permet de réduire la bande passante requise tout en maintenant une bonne synchronisation. Il est souvent utilisé dans les liaisons Ethernet 100BASE-TX.

### 2B1Q :
- **Utilisation** : 2B1Q est utilisé dans les communications DSL (Digital Subscriber Line) pour transmettre deux bits par symbole, ce qui permet de doubler le taux de transmission par rapport à NRZ. Il est utilisé pour maximiser le débit de données sur les lignes téléphoniques.

### RZ unipolaire/bipolaire :
- **Utilisation** : RZ est utilisé dans les communications par radio et les communications sans fil, où les transitions de signal sont nécessaires pour assurer la synchronisation.

### Miller :
- **Utilisation** : Le codage de Miller est utilisé dans les communications optiques, y compris les réseaux de fibre optique, car il maintient un niveau de signal constant pendant la transmission de données, ce qui facilite la détection du signal.

### CMI (Coded Mark Inversion) :
- **Utilisation** : CMI est utilisé dans diverses applications de communication, y compris les liaisons T1/E1, pour minimiser la composante DC du signal et faciliter la synchronisation des bits.
